Objetivo del Proyecto

El objetivo principal era construir una aplicación de streaming en tiempo real que funcionara sin la necesidad de un marco complejo como Spring Boot, enfocándome en las bases de Java y en la simplicidad de WebSocket. Esta decisión no solo reduce la complejidad del proyecto, sino que también mejora el rendimiento al prescindir de configuraciones adicionales.

Arquitectura del Proyecto

Servidor WebSocket:

La aplicación incluye un servidor WebSocket desarrollado con la API de Java para gestionar conexiones de transmisión de múltiples clientes. Este servidor recibe los datos de video y los transmite a todos los clientes conectados.

Cliente WebSocket para Transmisión:

El cliente de streaming, implementado también en Java, utiliza JavaCV para capturar video de la cámara. Esta captura se convierte en datos binarios y se envía a través del WebSocket hacia el servidor para su retransmisión en tiempo real.
